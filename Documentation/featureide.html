Description of the Contribution

In this section, we briefly describe the integration of our complex-constraint elimination algorithm.

Installtion

FeatureIDE comes in pre-packaged Eclipse versions for Windows that can be downloaded from its official website: http://wwwiti.cs.uni-magdeburg.de/iti_db/research/featureide/#download. Our algorithm was implemented in v3.1.0. The current version is v3.3.0. Currently, there exist only a pre-packaged version for Windows. If you would like to use FeatureIDE with Linux or Mac OS, download the source code from https://github.com/FeatureIDE/FeatureIDE and import the plug-in projects into your Eclipse (needs EMF). Then start a new Eclipse instance from the core.fm package.

Using the algorithm

Right-click on your feature model and select FeatureIDE -> product-equivalent model with simple constraints. The following dialog appears:

The dialog shows how many strict-complex and pseudo-complex constraint a feature model has. You can select a refactoring strategy (i.e., CNF, NNF, or a combination of both such that the minimal number of features and constraints are added). You can also preserve the number of configurations. In this case, for each additional requires constraint A => B, the requires constraint B => A is added. This is useful, when the target tool does not know how to handle abstract features and the number of products is important.
